Guía de integración Frontend → Backend

Base URL de la API: http://localhost:8080/api

Autenticación / Usuarios

GET /api/users - Lista de usuarios

GET /api/postulantes/ci/{ci} - Obtener postulante por CI

Postulantes

POST /api/postulantes

{
  "ci": "1234567",
  "nombres": "Juan",
  "apellidos": "Perez",
  "email": "juan@example.com",
  "telefono": "099123456"
}

Carreras

GET /api/carreras - Lista de carreras

Documentos

GET /api/postulantes/{postulanteId}/documentos - Lista documentos de un postulante

POST /api/postulantes/{postulanteId}/documentos/{documentoId}/upload - Subir archivo (body: binary, Content-Type: application/octet-stream)

Resultados

GET /api/postulantes/{postulanteId}/resultado - Obtener resultado JSON

GET /api/postulantes/{postulanteId}/resultado/pdf - Descargar PDF

Notas

  • Todos los endpoints devuelven y aceptan JSON salvo donde se indica "binary".
  • Si el frontend utiliza `HttpClient` de Java, mapear JSON con Jackson (`ObjectMapper`).
  • En el flujo de login, el frontend debe obtener y almacenar `User.id` para operaciones de documentos/resultados.
